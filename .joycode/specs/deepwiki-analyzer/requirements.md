# 需求文档 - DeepWiki 代码仓库分析工具

## 介绍

DeepWiki Analyzer 是一个基于 AI 的代码仓库分析工具，能够自动将 Gitee 公开代码仓库整理为结构化的 Markdown 文档。该工具通过 Git Clone 获取代码，使用 LangChain 驱动的大模型进行智能分析，最终生成包含架构图、流程图、数据库设计等完整内容的技术文档。

## 需求

### 需求 1 - 仓库代码获取

**用户故事：** 作为一个用户，需要输入 Gitee 仓库 URL 并自动获取代码，以便开始分析流程。

#### 验收标准

1. 当用户输入有效的 Gitee 公开仓库 URL 时，系统能够自动 Clone 仓库到临时目录。
2. 当仓库 Clone 成功后，系统返回仓库基本信息（名称、描述、主要语言、文件数量）。
3. 当输入无效 URL 时，系统返回明确的错误提示。
4. 当仓库为私有仓库时，系统提示"仅支持公开仓库"。
5. 分析完成后，临时目录应被自动清理。

---

### 需求 2 - 目录结构分析

**用户故事：** 作为一个用户，需要查看仓库的目录结构分析，以便了解项目的整体组织方式。

#### 验收标准

1. 生成的文档包含完整的目录树结构图（使用 ASCII 或 mermaid 展示）。
2. 分析每个主要目录的用途和职责。
3. 识别并标注入口文件（如 index.js、main.go、app.py 等）。
4. 识别配置文件目录（如 config/、settings/）。
5. 识别资源文件目录（如 assets/、public/、static/）。

---

### 需求 3 - 技术架构分析

**用户故事：** 作为一个用户，需要了解项目的技术栈和架构设计，以便快速把握项目技术选型。

#### 验收标准

1. 自动识别项目使用的编程语言。
2. 从 package.json、requirements.txt、go.mod、pom.xml 等文件解析依赖信息。
3. 识别使用的框架（React、Vue、Express、Spring Boot 等）。
4. 生成技术架构图（使用 mermaid graph TB 展示）。
5. 分析架构模式（MVC、微服务、单体应用、前后端分离等）。

---

### 需求 4 - 开发环境分析

**用户故事：** 作为一个开发者，需要知道如何搭建开发环境，以便快速参与项目开发。

#### 验收标准

1. 列出 Node.js、Python、Java 等运行环境的版本要求。
2. 列出必要的开发工具（数据库、Redis、消息队列等）。
3. 解析环境变量配置（.env.example 或文档中的环境变量说明）。
4. 提供启动命令（开发模式、生产模式）。
5. 说明数据库初始化步骤（如有）。

---

### 需求 5 - 功能模块分析

**用户故事：** 作为一个用户，需要了解系统的核心功能模块及其工作流程，以便理解业务逻辑。

#### 验收标准

1. 自动识别并分类核心功能模块（如用户模块、订单模块、支付模块）。
2. 每个模块提供功能描述和职责说明。
3. 核心模块生成 mermaid 流程图展示业务流程。
4. 标注模块间的依赖关系和调用链。
5. 识别 API 接口并分类展示（RESTful 路由、GraphQL 等）。

---

### 需求 6 - 数据库结构分析

**用户故事：** 作为一个用户，需要了解数据库表结构和表关系，以便理解数据模型设计。

#### 验收标准

1. 解析 SQL 文件、ORM 模型文件或 Migration 文件。
2. 列出所有数据库表及其字段说明。
3. 标注主键、外键、索引信息。
4. 生成 mermaid ER 图展示表关系。
5. 对于 NoSQL 项目，分析集合结构和索引设计。

---

### 需求 7 - 答辩问题生成

**用户故事：** 作为一个学生，需要获取针对该项目的答辩问题和参考答案，以便准备项目答辩。

#### 验收标准

1. 生成 10-15 个常见的答辩问题。
2. 问题涵盖技术选型、架构设计、功能实现、性能优化等方面。
3. 每个问题提供详细的参考答案。
4. 答案基于项目实际代码和技术栈生成。
5. 包含"为什么选择 X 而不是 Y"类对比问题。

---

### 需求 8 - Web 界面

**用户故事：** 作为一个用户，需要通过 Web 界面操作工具，以便无需命令行知识即可使用。

#### 验收标准

1. 首页提供仓库 URL 输入框和分析按钮。
2. 显示分析进度条（分为多个阶段：代码获取、目录分析、架构分析等）。
3. 分析完成后可直接预览生成的 MD 文档。
4. 提供下载按钮，支持下载 MD 文件。
5. 界面简洁美观，支持响应式布局。

---

### 需求 9 - 异步处理机制

**用户故事：** 作为一个用户，希望分析过程在后台进行，以便不会因为等待超时而失败。

#### 验收标准

1. 分析任务提交后立即返回任务 ID。
2. 前端通过轮询获取分析进度。
3. 支持多个分析任务并行处理。
4. 任务失败时提供详细的错误信息。
5. 支持任务状态查询（pending、processing、completed、failed）。

---

### 需求 10 - 配置管理

**用户故事：** 作为一个用户，需要通过配置文件管理 API 密钥和模型设置，以便灵活切换不同 AI 服务。

#### 验收标准

1. 支持 .env 文件配置。
2. 配置项包括：OPENAI_API_KEY、OPENAI_BASE_URL、MODEL_NAME。
3. 所有敏感信息仅从环境变量读取，不硬编码。
4. 启动时验证必要配置项是否存在。
5. 配置缺失时给出明确的错误提示。
